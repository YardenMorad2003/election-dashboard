# Israeli Elections Data — MCP server

A [Model Context Protocol](https://modelcontextprotocol.io) server that lets
Claude (Desktop / Code / claude.ai) and ChatGPT query this repository's
election dataset directly:
locality-level Knesset results 1992–2022 (K13–K25), the 1996/1999/2001 direct
prime-ministerial contests, party histories, vote-transfer matrices
(ecological inference + INES surveys), and CBS socio-economic profiles.

Every tool returns a small, question-shaped slice (typically 0.5–3 KB) — the
server never dumps whole data files into the model's context. Locality and
party arguments accept **Hebrew or English** names (or semel / ballot codes).

Two implementations share the same tool surface:

- **`server.py`** — local stdio server (Claude Code / Claude Desktop).
- **`worker/`** — remote Cloudflare Worker (MCP Streamable HTTP) for
  claude.ai custom connectors and ChatGPT, fetching the same JSON from the
  GitHub Pages CDN.

## Tools

| Tool | What it answers |
|---|---|
| `list_elections()` | Which elections exist; national bloc shares + turnout per election |
| `search_locality(query)` | Resolve a locality by Hebrew/English name or semel |
| `get_locality_results(locality, knesset?)` | Bloc breakdown + turnout for one place, one or all elections |
| `get_locality_parties(locality, knesset)` | Party-level shares (% of valid votes) + estimated votes in one locality |
| `get_national_results(knesset)` | Every party's votes, share, seats, bloc |
| `get_party(party, knesset?)` | A party's 1992–2022 trajectory + strongholds |
| `get_vote_transfers(from_k, to_k, source)` | Bloc transfer matrix, `ballot` (ecological) or `survey` (INES) |
| `get_pm_direct(year, locality?)` | 1996/1999/2001 direct-PM result, national or local |
| `get_demographics(locality)` | CBS 2021 socio-economic profile + religious composition |

## Setup

Requires Python 3.10+ and the MCP SDK:

```
pip install -r mcp/requirements.txt
```

### Claude Code

```
claude mcp add israeli-elections -- python "C:\Users\yarde\Downloads\election-dashboard-extracted\election-dashboard-main\mcp\server.py"
```

(Add `-s user` to make it available in every project. Remove with
`claude mcp remove israeli-elections`.)

### Claude Desktop

Add to `%APPDATA%\Claude\claude_desktop_config.json` under `mcpServers`:

```json
{
  "mcpServers": {
    "israeli-elections": {
      "command": "python",
      "args": ["C:\\Users\\yarde\\Downloads\\election-dashboard-extracted\\election-dashboard-main\\mcp\\server.py"]
    }
  }
}
```

Restart Claude Desktop; the tools appear under the 🔌 connector icon.

## Remote server (claude.ai + ChatGPT) — Cloudflare Worker

`worker/src/index.mjs` is a dependency-free Worker speaking MCP Streamable
HTTP (stateless JSON-RPC over `POST /mcp`). It serves the same nine tools,
plus `search` / `fetch` aliases over localities so ChatGPT deep research can
use it too. No auth — the data is public; anyone with the URL can query it.

Instead of parsing the big dashboard files per request (localities.json is
4.3 MB), the Worker reads pre-sliced artifacts from `data/mcp/` on the CDN
— a slim locality index, one small file per locality, per-Knesset party
shards — generated by `python mcp/build_worker_data.py`. **Re-run that
script and commit `data/mcp/` whenever the source data files change.**
Small source files (core, names_en, parties_national, vote_transfers,
survey_transfers, pm_direct) are fetched as-is. Everything stays within
Cloudflare's free tier (CDN responses are edge-cached for 24 h).

### Deploy

```
cd mcp/worker
npx wrangler login     # one-time OAuth into your Cloudflare account
npx wrangler deploy
```

The deploy prints the URL, e.g.
`https://israeli-elections-mcp.<your-subdomain>.workers.dev` — the MCP
endpoint is that URL plus `/mcp`. Note the Worker reads the **published**
GitHub Pages data, so push data changes before/with deploying.

Local dev (`npx wrangler dev`) works against the live CDN, or against local
data via
`npx wrangler dev --var "DATA_BASE:http://127.0.0.1:8788/data/"` with an
HTTP/1.1 static server on 8788 (plain `python -m http.server` speaks
HTTP/1.0, which workerd's fetch drops mid-body).

### claude.ai (web / mobile / desktop)

Settings → Connectors → **Add custom connector** → paste
`https://<worker-url>/mcp`. No OAuth to configure.

### ChatGPT

Settings → Connectors → Advanced → enable **Developer mode**, then add a
custom connector with the same `/mcp` URL (no authentication). In deep
research, the connector works through the `search` / `fetch` tools.

### Claude Code (remote instead of local)

```
claude mcp add -s user -t http israeli-elections-remote https://<worker-url>/mcp
```

## Data & method notes

- Blocs: right, haredi, center, left, arab; `right_haredi` = right + haredi.
  From K21 (April 2019), `opposition_right` (Yisrael Beiteinu) is reported
  separately and counted inside `center_left_arab`.
- Ballot transfers: constrained least-squares ecological inference over
  ~1,000 localities, with abstention flows; survey transfers: INES
  recalled-vote crosstabs (21→22 is a true panel).
- Sources: Central Elections Committee official locality results; CBS
  socio-economic municipal data (2021); Israel National Election Studies.
- Interactive dashboard: https://yardenmorad2003.github.io/election-dashboard/

The stdio server reads `../data/*.json` — the same files the site fetches —
so regenerating dashboard data automatically updates it. The Worker reads
the published CDN copies plus the derived `data/mcp/` artifacts; after a
data change, re-run `python mcp/build_worker_data.py` and push.
